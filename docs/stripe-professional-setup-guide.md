# Stripe 設定ガイド — Professional 即時購入（秋山さん向け）

最終更新: 2026-06-11 ／ 作成: クロちゃん（Claude Code）

このガイドは **Professional ライセンス（B表）をオンラインで即時購入できるようにする** ための、Stripe ダッシュボード作業の手順書です。
画面のボタン名・入力値まで書いてあるので、上から順にそのまま進めてください。**まずは必ず「テストモード」から**始めます。本番（実際の課金）は最後の章で切り替えます。

> 店舗BGMパス（A表・年額サブスク）は「後追い」で別フェーズに実装します。本ガイドの対象は **B表（単曲買い切り）** です。A表の概要は最後の付録に記載しました。

---

## 0. 用語ミニ辞典（最初に目を通すだけでOK）

| 用語 | 一言でいうと |
|---|---|
| **Product（商品）** | 売り物の入れ物。例:「Professional ライセンス ¥5,500」 |
| **Price（価格）** | 商品につける値段。1つごとに `price_xxxx` という **Price ID** が発行される。**コードはこのIDで商品を呼び出す**ので、作ったら必ず控えてクロちゃんに渡してください |
| **One-time（一括）** | 1回きりの支払い（買い切り）。B表はすべてこれ |
| **Recurring（継続）** | 毎月・毎年の自動課金（サブスク）。A表の店舗パスで使う（後追い） |
| **テストモード / 本番モード** | テストは偽の決済で練習する場所。本番は実際にお金が動く場所。**APIキーもPrice IDも、テストと本番で完全に別物**です |
| **Webhook（ウェブフック）** | 「決済が完了した」等のお知らせを Stripe が自動でサーバーへ送る仕組み。すでに設定済みなので今回は基本さわりません |
| **テスト用カード番号** | `4242 4242 4242 4242`（有効期限＝未来の任意の日付、CVC＝任意の3桁、郵便番号＝任意）。テストモードでの決済練習に使う |

## 1. 作る商品の一覧（これだけ作ればOK）

**ポイント：用途は11種類ありますが、値段は5種類しかありません。**
なので **商品は「値段ごとに5つ」だけ** 作ります。どの用途で買ったか（映画祭/広告/TV など）は、購入フォームの選択がそのまま証明書とStripeの記録（metadata）に残るので、商品を11個も作る必要はありません。

| # | 商品名（この通りに入力） | 価格 | 種別 | 後でクロちゃんに渡す値 |
|---|---|---|---|---|
| 1 | `H/MIX Professional License (¥3,300)` | ¥3,300 | One-time | Price ID |
| 2 | `H/MIX Professional License (¥4,400)` | ¥4,400 | One-time | Price ID |
| 3 | `H/MIX Professional License (¥5,500)` | ¥5,500 | One-time | Price ID |
| 4 | `H/MIX Professional License (¥16,500)` | ¥16,500 | One-time | Price ID |
| 5 | `H/MIX Professional License (¥27,500)` | ¥27,500 | One-time | Price ID |

> 通貨はすべて **JPY（日本円）**。消費税は価格に含む（税込）ので、税設定は触らなくてOKです。

---

## 2. テストモードで商品を1つ作る（手順）

**目的：** ¥3,300 の商品を1つ作り、Price ID を取得する。同じ手順を5回くり返します。

1. ブラウザで Stripe ダッシュボード（https://dashboard.stripe.com/）を開いてログイン
2. 画面 **右上のトグルが「テストモード」になっていることを確認**（オレンジ色の「テスト環境」表示が出ます）。なっていなければクリックして切り替える
3. 左メニューの **「商品カタログ（Product catalog）」** をクリック
4. 右上の **「＋商品を追加（Add product）」** をクリック
5. **名前（Name）** に `H/MIX Professional License (¥3,300)` と入力
6. 説明（Description）は空欄でOK（任意）
7. 「料金体系（Pricing）」で：
   - 金額: **3300**
   - 通貨: **JPY**
   - **「一括（One-time）」** を選択（「継続（Recurring）」ではない！）
8. 右上の **「商品を保存（Save product）」** をクリック
9. 保存後の商品ページで、**価格（Price）欄の `price_` で始まるIDをコピー**して、メモ帳に「¥3,300 = price_xxxxx」と控える

> ✅ 完了の目安：商品一覧に「H/MIX Professional License (¥3,300)」が表示され、Price ID（`price_…`）が手元に控えられている状態。

**この手順1〜9を、残り4つ（¥4,400 / ¥5,500 / ¥16,500 / ¥27,500）でもくり返してください。**

---

## 3. メール領収書の自動送信をONにする

1. 左下の **「設定（Settings）」**（歯車アイコン）→ **「メール（Emails）」** を開く
2. **「支払い後の領収書（Successful payments / Email customers about ...）」** を **ON**
3. （インボイス対応がある場合）「請求書（Invoices）」設定で、事業者名 `秋山裕和 / H/MIX GALLERY` と **インボイス登録番号（T+13桁）** を表示する欄があれば入力

> ❓ インボイス登録番号をお持ちかどうか教えてください。法人のお客様（企業VP ¥16,500 など）が経費精算で必要にすることがあります。未登録なら今は空欄でOKです。

# 店舗BGMパス（A表・年額サブスク）設定ガイド

最終更新: 2026-06-14

店舗BGMパスは **年額サブスク（継続課金）** です。B表（買い切り）と違い「自動更新」「解約」があるので設定が少し増えますが、商品作成は**スクリプトが自動でやってくれる**ので手作業はほとんどありません。

> コード側（購入フォーム・証明書・バックエンド）は実装済みです。あとは ①商品を作る ②Price ID を教えてもらう ③カスタマーポータルをONにする の3つで開通します。

## A-1. 作る商品（スクリプトが自動作成）

| 商品 | 価格 | 種別 |
|---|---|---|
| 店舗BGMパス（1店舗目） | ¥9,900/年 | 継続（Recurring・年次） |
| 追加店舗（2店舗目以降・1店舗） | ¥4,950/年 | 継続（Recurring・年次） |
| 施設パス（館内BGM） | ¥19,800/年 | 継続（Recurring・年次） |

「3店舗まとめて」は **店舗パス×1＋追加店舗×2** という形で自動計算されます（¥9,900＋¥4,950×2＝¥19,800）。

## A-2. 商品をスクリプトで一括作成（まずテスト）

B表のときと同じやり方です。PowerShell で：

```powershell
cd C:\Users\hiro\projects\hmix-stripe
$env:STRIPE_SECRET_KEY = 'sk_test_あなたのテストキー'
node scripts/create-store-products.js
```

実行すると、画面の最後に **テスト用 Price ID が3つ** 出ます。

---

## 📌 Price ID とは？ どこで手に入れる？（詳しく）

**Price ID** は、Stripe が価格1つごとに自動で発行する `price_` で始まる文字列です（例: `price_1ThsoW33JkAcnXC9VEBXjs7h`）。
H/MIX のコードは「¥9,900の商品」を名前ではなく**この ID で呼び出す**ので、商品を作ったら必ず ID を控えてクロちゃんに渡す必要があります。

> ⚠️ **テストと本番で別物です。** テストモードで作った Price ID（テスト用）と、本番モードで作った Price ID（本番用）は完全に違う文字列になります。取り違えると決済できません。今は**テストモードの ID** を取得します。

Price ID の取り方は2通りあります。**方法①（スクリプト）が断然ラク**です。

### 方法①：スクリプトの出力からコピー（推奨・手作業ほぼゼロ）

上の `node scripts/create-store-products.js` を実行すると、画面の最後にこういう表示が出ます：

```
================ 結果（この値を控えてください） ================
store_base  ¥9,900/年  = price_1AbcdE...
store_add   ¥4,950/年  = price_1FghiJ...
facility    ¥19,800/年 = price_1KlmnO...

--- functions/index.js 用スニペット（TEST） ---
const STORE_PRICES_TEST = {
  store_base: "price_1AbcdE...",  // ¥9,900/年
  store_add:  "price_1FghiJ...",  // ¥4,950/年
  facility:   "price_1KlmnO...",  // ¥19,800/年
};
==============================================================
```

この `price_…` の3行（または `const STORE_PRICES_TEST = {…}` のブロックごと）を**そのままコピーしてクロちゃんに貼る**だけで完了です。
※ このスクリプトは何度実行しても同じ商品を再利用するので、ID を取り損ねたら**もう一度実行すれば同じ ID がまた表示されます**（重複作成されません）。

### 方法②：Stripe ダッシュボードの画面から手でコピー（スクリプトを使わない場合）

1. Stripe ダッシュボード（https://dashboard.stripe.com/）にログイン
2. 画面**右上のトグルが「テストモード」**になっていることを確認（オレンジの「テスト環境」表示）
3. 左メニュー **「商品カタログ（Product catalog）」** → 一覧から **「H/MIX 店舗BGMパス（1店舗・年額）」** をクリック
4. 商品の詳細ページを下にスクロールすると **「価格（Pricing）」** という表があります
5. その価格の行に `¥9,900 / year` と並んで、**`price_` で始まる薄いグレーの文字列**が表示されています。多くの場合その文字列に**マウスを乗せるとコピーアイコン📋**が出るので、それをクリックするとコピーできます
   - 見当たらない場合：価格行の右端の **「…」メニュー → 「価格IDをコピー（Copy price ID）」** を選ぶ
6. 同様に **「追加店舗（¥4,950）」「施設パス（¥19,800）」** の商品ページでもそれぞれの `price_…` をコピー

> 💡 商品ID（`prod_…`）と価格ID（`price_…`）は別物です。**必要なのは `price_` の方**です（`prod_` ではありません）。

> ✅ 完了の目安: `price_…` の文字列が **3つ**（store_base / store_add / facility）手元に揃っている。テストモードで取得したものであること。

## A-3. カスタマーポータルを有効化（お客様が自分で解約できるように）

サブスクは「お客様が自分で解約・カード変更できる窓口」を用意するのがルールです。

1. Stripeダッシュボード左下の **「設定（Settings）」**（歯車）→ **「Billing」→「カスタマーポータル（Customer portal）」** を開く
2. **「サブスクリプションのキャンセルを許可（Cancel subscriptions）」** を ON
3. **「支払い方法の更新を許可（Update payment methods）」** を ON
4. ページ上部の **「保存（Save）」/「リンクを有効化（Activate link）」** をクリック

> ❓ これは「テストモード」「本番モード」それぞれで1回ずつONにする必要があります。まずはテストモードでON。

## A-4. 更新失敗時の自動リトライ（Smart Retries）を確認

カード期限切れなどで年次更新が失敗したとき、自動で数回リトライさせます。

1. **「設定」→「Billing」→「自動課金（Subscriptions and emails / Manage failed payments）」** を開く
2. **「Smart Retries（スマートリトライ）」** が ON になっていることを確認（通常は初期状態でON）

## A-5. Webhook に解約イベントを追加

解約されたら証明書を失効扱いにするため、サーバーが解約通知を受け取れるようにします。

1. **「開発者（Developers）」→「Webhook」** で、すでに登録済みのエンドポイント（`stripeWebhook`）を開く
2. **「リッスンするイベントを更新（Update / Select events）」** で、既存の `checkout.session.completed` に加えて **`customer.subscription.deleted`** を追加して保存
3. これも **テスト／本番それぞれ** で行う

## A-6. 作業後にクロちゃんへ共有してほしいもの

```
【店舗BGMパス テスト用 Price ID】
store_base (¥9,900/年)  = price_________
store_add  (¥4,950/年)  = price_________
facility   (¥19,800/年) = price_________

カスタマーポータル: テストでON済み（はい / まだ）
Webhookに customer.subscription.deleted を追加: 済み（はい / まだ）
```

これが揃ったら、テストモードで「店舗パス購入 → 証明書発行 → カスタマーポータルで解約」までe2e検証します。

## A-7. テスト → 本番 切替（検証OKのあと）

1. 本番モードで `node scripts/create-store-products.js --live`（制限付きキー `rk_live_` 推奨）を実行 → **本番 Price ID** を取得
2. クロちゃんが `STORE_PRICES_LIVE` に登録 ＋ フロントの `STORE_PASS_ENABLED = true` に変更してデプロイ
3. 本番モードでもカスタマーポータルON・Webhookに `customer.subscription.deleted` 追加を確認
4. 実カードで店舗パス ¥9,900 を1件 → カスタマーポータルですぐ解約（年次更新前なので追加課金なし）して動作確認

> `STORE_PASS_ENABLED` が `true` になるまで、サイト上の店舗BGMパスは「お問い合わせで受付中」のまま表示され、誤購入は起きません。
