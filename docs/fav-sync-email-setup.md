# お気に入りクラウド同期 スライス2 — メールリンク有効化ガイド（秋山さん向け・1分）

端末間同期（PC↔スマホで同じ音楽手帖）は、コードは実装・本番反映済みです。
あと **Firebase Console で「メールリンク（パスワードなし）ログイン」を ON にするだけ** で開通します。

## 手順（Firebase Console）
1. https://console.firebase.google.com/ → プロジェクト **hmix-footprints** を開く
2. 左メニュー **Authentication** → 上タブ **Sign-in method**
3. **「ログイン プロバイダを追加」**（または既存の「メール / パスワード」を開く）
4. **「メール / パスワード」を有効化**し、その中の
   **「メールリンク（パスワードなしのログイン）」を ON** にして保存
   - パスワード自体は使いません。メールリンクだけ ON でOK。
5. 完了。**認可ドメインは設定済み**（`www.hmix.net` / `hmix.net` を確認済み）なので追加不要。

## 動作（有効化後）
- 音楽手帖の右上「**別の端末でも使う**」→ メール入力 → リンク送信。
- 届いたメールのリンクを開くと、パスワードなしで同期開始。
  - 匿名のまま使っていた端末は `linkWithCredential` で**同じデータのままアカウント昇格**。
  - 別端末で同じメールでログインすると、その端末のお気に入りも**LWWマージ**で統合。
- セキュリティ: 別端末でリンクを開く時は、本人確認のためメール再入力を促します（Firebase標準）。

## コスト
メールリンク認証は無料（有料は電話SMS認証のみ・未使用）。送信メールはFirebase標準枠。

## 確認テスト（有効化後）
1. PCの手帖で「別の端末でも使う」→自分のメール宛に送信→PCで届いたリンクを開く→「同期した」トースト。
2. スマホで同じ曲ページを開き、手帖→「別の端末でも使う」→同じメール→リンク→お気に入りが揃う。

実装詳細: `assets/js/fav-sync.js`（`HMIX_FAV_SYNC.startEmailSync/openSync/account/signOutSync` ＋ 復帰時 `isSignInWithEmailLink`→`linkWithCredential`/`signInWithEmailLink`）。
